# 年賀状住所更新バッチインポート用 仕様書（AI向け）

あなたはこれから、年賀状（または喪中はがき等）の宛名面の写真を渡されます。各写真から宛先の情報を読み取り、`address_updates` というJSON配列の形式で出力してください。

## やること

1. 渡された写真を1枚ずつ確認し、宛名面に書かれている情報を読み取る
2. 宛先1件につき `address_updates` 配列の1要素を作る（同じ宛先の写真が複数枚（封筒の表裏、撮り直し等）に分かれていても、1要素にまとめること）
3. 読み取れない・自信が持てない箇所は、無理に埋めずに `_comment` に書く（後述）
4. 最終的に `{"address_updates": [...]}` という1つのJSONオブジェクトを出力する（前後に説明文をつけず、JSONのみ、または明確にコードブロックで囲って出力すること）

## 出力フォーマット

各要素のフィールドは以下の通り。

| フィールド | 必須 | 説明 |
|---|---|---|
| `match_hint` | ほぼ必須 | 宛名から読み取った氏名・カナ。`last_name`/`first_name`/`last_name_kana`/`first_name_kana` を持つオブジェクト。**これは「この葉書の宛名に書かれている氏名そのもの」**であり、データベースに登録されている氏名と一致するかどうかはアプリ側が判定するので、あなたは気にしなくてよい |
| `name_update` | 任意 | 通常は省略してよい。ユーザーから「この人物は以前と姓が変わっている（結婚・離婚等）」と明示的に伝えられた場合のみ、新しい氏名をここに入れる。`honorific`（敬称。例: 様、先生）も同じオブジェクトに入れてよい。現在の登録と異なる敬称が宛名面から読み取れる場合（例: 「様」ではなく「先生」と書かれている）のみ入れ、読み取れない・変更が無さそうな場合は省略する |
| `classification` | 任意（既定`自宅`） | 住所の種別。会社の住所だと判断できる場合は `勤務先` にする。それ以外は省略してよい |
| `zip` | **必須** | 郵便番号。`123-4567`のようにハイフンを入れて読み取る（読みにくい場合は`_comment`に書き、推測でよいので埋める） |
| `prefecture` | 任意 | 都道府県 |
| `city` | 任意 | 市区町村（「〜区」「〜市」「〜郡〜町」など） |
| `block` | 任意 | 町名・丁目・番地など（市区町村より後、建物名より前の部分。例: `上大崎2-24-9`） |
| `building` | 任意 | マンション名・建物名・部屋番号など |
| `companions` | 任意 | 連名（同じ葉書に複数名前が書かれている場合）。配列。各要素は `last_name`/`first_name`/`last_name_kana`/`first_name_kana`/`honorific`/`birthday`/`gender` を持つオブジェクト。**姓が宛名本人と同じ場合、連名側の`last_name`は省略してよい**（日本の年賀状の慣習として、同姓の場合は名のみ書かれることが多い）。敬称（様・くん・ちゃん等）が書かれていればそのまま`honorific`に入れる |
| `_comment` | 任意 | 読み取りに自信が持てない箇所、判断に迷った点などを自由記述。データベースには保存されず、人間が確認する際の参考情報としてのみ使われる |
| `_confidence` | 任意（既定`1.0`） | 0.0〜1.0の数値。文字が綺麗な印刷で迷いなく読めた場合は1.0、手書きで一部読みにくい・崩し字・滲み等がある場合はそれに応じて下げる（目安: 0.8未満だと確認画面で警告表示される） |

## 注意事項

- 住所は「郵便番号」「都道府県」「市区町村」「町名番地」「建物名」の5項目に分けて読み取ること。住所をひと続きの文字列のまま `prefecture` や `block` に詰め込まないこと
- 氏名は「姓」と「名」を分けること。スペースや改行で区切られていることが多い
- このタスクでは既存の住所録データは渡されない。照合・重複チェックはアプリ側で行うので、写真から読み取れる情報だけを正確に転記することに集中してよい（誰の住所かを判断する必要はない）
- `person_id` のようなデータベース内部のIDは、あなたが知る・指定する必要は一切ない
- 出力は妥当なJSON（末尾カンマなし、ダブルクォート使用、コメントなし）にすること

## 出力例

```json
{
  "address_updates": [
    {
      "match_hint": {
        "last_name": "山田", "first_name": "太郎",
        "last_name_kana": "ヤマダ", "first_name_kana": "タロウ"
      },
      "classification": "自宅",
      "zip": "123-4567",
      "prefecture": "東京都", "city": "品川区", "block": "上大崎2-24-9", "building": "アイケイビル1F",
      "companions": [
        { "first_name": "花子", "honorific": "様" }
      ],
      "_comment": "郵便番号の最後の桁が滲んでいて読み取りにくい",
      "_confidence": 0.75
    }
  ]
}
```
